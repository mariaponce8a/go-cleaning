## 1. ℹ️ Información General del Proyecto
- **📌 Nombre del Proyecto:** Sistema de control Go Cleaning  
- **🏷️ Versión de Línea Base:** BL-1.0  
- **🎯 Propósito:** Manejo completo de órdenes de limpieza, desde registro de clientes hasta entrega final.  

**👥 Roles GCS**

- 🔐 Aprobador CAB: Valida cambios críticos

**🧩 Esquema de Versionado:** SemVer (`vMAJOR.MINOR.PATCH`)  
- 🔴 **MAJOR:** Cambios incompatibles  
- 🟡 **MINOR:** Nuevas funcionalidades compatibles  
- 🟢 **PATCH:** Correcciones de errores  

**🌳 Estrategia de Ramas (GitFlow adaptado):**  
- `main` → 🚀 Líneas base estables  
- `develop` → 🔄 Integración continua  
- `feature/*` → ✨ Nuevas funcionalidades  
- `hotfix/*` → 🛠️ Correcciones urgentes

## 2. 🏗️ Arquitectura del Sistema
El sistema implementa **arquitectura de tres capas**:

- 🎨 **Capa de Presentación:** Angular 17+, TypeScript, Angular Material, Ionic  
- ⚙️ **Capa de Lógica de Negocio:** API RESTful en PHP con Framework Flight  
- 🗄️ **Capa de Datos:** MySQL con procedimientos almacenados
